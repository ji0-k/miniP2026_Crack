[ 수동 배포 및 파이썬 우회 가이드 ]

서버 컴퓨터의 보안 정책상 파이썬 설치(Installer)가 거부될 경우, 아래 순서대로 진행해 주세요.

1. 아래 링크를 복사하여 브라우저 주소창에 넣고 다운로드 받으세요. (무설치 포터블 버전)
   https://www.python.org/ftp/python/3.10.11/python-3.10.11-embed-amd64.zip

2. 프로젝트 폴더(app.py가 있는 폴더) 안에 [ python_portable ] 이라는 새 폴더를 만드세요.

3. 다운로드 받은 ZIP 파일의 압축을 풀어 [ python_portable ] 폴더 안에 모든 파일을 넣으세요.

4. 이제 [ ONE_CLICK_START.bat ] 파일을 실행하면 됩니다.

[ 프로젝트 초기 설정 가이드 ]
깃에서 코드를 처음 받은 후, 보안상 제외된 'secrets' 폴더를 직접 설정해야 서버가 작동합니다.

1. 프로젝트 루트에 [ secrets ] 폴더를 새로 만듭니다.
2. [ secrets.example ] 폴더의 모든 파일을 복사해서 [ secrets ] 폴더 안에 넣습니다.
3. 복사한 파일들의 이름에서 [ .example ] 확장자를 지웁니다.
   (예: .env.example -> .env / kakao_js_key.txt.example -> kakao_js_key.txt)
4. [ .env ] 파일을 열어 본인의 DB 접속 정보와 API 키를 입력합니다.

[ 💡 파이참(PyCharm) 사용자를 위한 설치 속도 개선 ]
1. **버전 고정**: `requirements.txt`의 버전을 고정하여 체크 속도를 높였습니다.
2. **인터프리터 설정**: 파이참 우측 하단에서 `Python Interpreter`를 클릭 후, 새 가상환경을 만들지 말고 **프로젝트 내부의 [.venv] 폴더**를 사용하도록 지정하세요.
3. **SourceTree 활용**: 'Download ZIP' 대신 소스트리의 **'Pull'** 기능을 사용하세요. 그러면 `.venv` 폴더가 삭제되지 않고 유지되어 라이브러리를 매번 새로 설치할 필요가 없습니다.

[ ⚠️ .git 폴더의 역할 ]
`.git` 폴더는 프로젝트의 모든 변경 이력을 담고 있는 '심장'입니다.
- **문제점**: 'Download ZIP'으로 받으면 이 폴더가 없습니다. 그래서 변경 사항만 가져오는(Pull) 기능이 안 되고 매번 새로 다운로드해서 덮어씌워야 합니다.
- **해결**: 반드시 `git clone`을 통해 프로젝트를 시작하세요. 그래야 소스트리에서 한 번의 클릭만으로 업데이트가 가능합니다.

* 주의: 반드시 '모든 파일의 압축 풀기'를 완료한 상태에서 폴더 안에서 실행해야 합니다. 
* 실행 시 검은 창이 바로 꺼지지 않고 어떤 오류인지 알려주도록 수정해 두었습니다.

🚀 이제 서버를 시작할 준비가 되었습니다!
