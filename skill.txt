프론트엔드 개발자 시각에서 이 PPT 인터페이스 기법 분석

1. 워크프레임 및 개념 명칭
이러한 프레젠테이션 기법은 **ZUI (Zooming User Interface)** 또는 **무한 캔버스 (Infinite Canvas)** 뷰어라고 불립니다. 
일반적인 슬라이드 방식(PowerPoint, Keynote)과 달리, 거대한 하나의 평면 위에 콘텐츠들을 여기저기 흩뿌려 놓고 카메라(뷰포트)가 특정 좌표(X, Y)로 이동하며 확대/축소(Zoom)하는 방식입니다. 가장 대표적인 상용 서비스로는 **Prezi(프레지)**가 있습니다.

2. 사용된 라이브러리 (Library & Framework)
현재 `ppt.html` 코드를 분석해보면 외부의 무거운 ZUI 프레임워크를 사용하지 않고 개발자 본인이 **순수 바닐라 JS(Vanilla JavaScript)와 CSS3** 만을 활용해 직접 구현(Custom Implementation)한 방식입니다. 

[참고: 만약 라이브러리를 쓴다면?]
보통 오픈소스로는 **impress.js** (가장 유명)나 **reveal.js** (3D 효과 등)를 많이 사용하지만, CRACK 프로젝트에서는 직관적인 제어를 위해 아래와 같은 자체 기술 요소를 사용했습니다.

3. 핵심 구현 기술 (Technical Elements)
- **CSS3 `transform`**: 각 슬라이드(`div.spot`)를 캔버스 상의 거대한 절대 좌표에 배치할 때 `absolute` 속성과 함께 `left`, `top`을 지정합니다.
- **CSS3 `transition`**: 부드러운 카메라 이동 효과를 위해 `transition: transform 2s cubic-bezier(...)`와 같은 감속 함수(Easing function)를 적용했습니다.
- **Vanilla JS `spotsCoord` 배열**: 
  `[{x: 0, y: 0, scale: 1}, {x: 2000, y: 0, scale: 0.8}, ...]` 처럼 각 스팟(슬라이드)마다 카메라가 이동할 중심 좌표와 줌 배율을 배열로 저장해두고,
  방향키(Keydown) 이벤트가 발생할 때마다 해당 좌표의 역방향으로 `#canvas` 전체를 통째로 `transform: translate()` 시키는 방식(마치 카메라가 이동하는 듯한 착시 효과)을 사용했습니다.
